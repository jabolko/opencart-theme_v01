8. Known Issues to Fix

A. Bootstrap loading state
OC uses .button('loading') which adds Bootstrap's loading text. Our CSS doesn't override this — button text changes to "Loading..." in English.

B. Focus ring inconsistent
Radio buttons have outline: none (we removed it for the blue dot fix) but CTA buttons have no focus style at all. Keyboard users can't see which element is active.

C. Confirm proxy button styling
The proxy #ck-confirm-proxy uses .btn.btn-primary class which picks up Bootstrap's blue + our gold override. But it's a <button> while others are <input> — different default rendering.

D. Loading spinner missing
Our proposed .is-loading::after spinner doesn't exist in current CSS. Bootstrap's .button('loading') just changes text value.
